Why: CocoaPods 0.39.0 (from 2016) cannot resolve modern podspecs or generate

## DESCRIPTION
The project was created with CocoaPods 0.39.0, which predates arm64 support and
modern podspec formats. The gem must be updated to the current stable 1.x release
before any pod operations can succeed on modern macOS/Xcode.

## ACCEPTANCE CRITERIA
- [x] CocoaPods gem is updated to version >= 1.15 (`pod --version` confirms)
- [x] `pod install` does not fail with a CocoaPods version error

      so stale cached artifacts from the 0.39.0 era are cleared

## IMPLEMENTATION DETAIL
Run the following to update CocoaPods:

  sudo gem install cocoapods

If the system Ruby is too old (macOS ships Ruby 2.6 which works, but Homebrew Ruby
is safer):

  brew install ruby
  export PATH="/opt/homebrew/opt/ruby/bin:$PATH"
  gem install cocoapods

After installing, verify:
  pod --version    # should print 1.15.x or newer

Remove stale artifacts before the next ticket runs pod install:
  rm -rf Pods/ Podfile.lock DNSSwitcher.xcworkspace
